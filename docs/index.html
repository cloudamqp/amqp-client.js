Set prefetch limit. Recommended to set as each unacknowledge message will be store in memory of the client. The server won't deliver more messages than the limit until messages are acknowledged.

Parameters
prefetchCount (number) number of messages to limit to
prefetchSize (any = 0) number of bytes to limit to (not supported by RabbitMQ)
global (any = false) if the prefetch is limited to the channel, or if false to each consumer
Returns
Promise<any>

Parameters
name (any = "") name of the queue, if empty the server will generate a name
params (Object = {})
Name Description
params.passive boolean (default false) if the queue name doesn't exists the channel will be closed with an error, fulfilled if the queue name does exists
params.durable boolean (default name!=="") if the queue should survive server restarts
params.autoDelete boolean (default name==="") if the queue should be deleted when the last consumer of the queue disconnects

Returns
